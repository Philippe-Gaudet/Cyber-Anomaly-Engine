BARCODE

Catching network attacks by the shape of the traffic.
A plain-language guide to every term in the project — written for a security analyst, no advanced math required.
Topological anomaly detection · CICIDS2017 · Read time ≈ 45–60 min

0 The one-paragraph version

Most intrusion detection looks at one connection at a time. Barcode does something different: it takes a group of network flows, treats them as a cloud of dots in space, and measures the shape of that cloud. Normal traffic has a characteristic shape. An attack bends it — a port scan pokes holes in it, a DDoS crushes it into a blob. Barcode learns the shape of normal traffic once, then raises an alert whenever a new batch of traffic is "shaped wrong." It backs this up with classic models and a graph neural network, and compares all of them honestly.

Why a security analyst should care
Signature and per-flow models miss attacks that only look abnormal in aggregate (slow scans, distributed floods, beaconing). Barcode is an unsupervised, behavior-based layer: it doesn't need a signature for the attack, only a sense of what "normal shaped" traffic looks like. That makes it useful against novel/zero-day patterns.

2 The math vocabulary (in plain English)

This is a branch of math called Topology — the study of shape and connectivity, the part that doesn't change when you stretch or bend something. The specific toolkit is Topological Data Analysis (TDA), and its workhorse is persistent homology. Don't worry about the names; here's what each one actually means.

Point cloud
Take one window of traffic. Each flow has ~22 numbers (duration, packet counts, byte rates…). Treat those 22 numbers as coordinates, so each flow becomes a single dot in a 22-dimensional space. A window of 50 flows is a cloud of 50 dots.
Security analogy: like plotting every connection as a star; an attack changes the constellation, not just one star.
Sliding window
Instead of scoring one flow, Barcode scores a window of 50 consecutive flows, then slides forward by 25 and scores the next overlapping batch. This is what lets it see aggregate behavior over time.
Security analogy: a rolling 50-event view on a SIEM, not a single log line.
Topology / "the shape"
The properties of the cloud that survive stretching: how many separate clumps it has, and how many loops (rings / holes) it has. Two clouds can have very different exact coordinates but the same topology.
Security analogy: like fingerprinting structure, robust to small jitter in the raw numbers.
Simplicial complex & simplex
To measure shape, you connect nearby dots. A simplex is just the building block: a point (0), a line between two points (1), a filled triangle between three (2), and so on. Gluing many simplices together gives a simplicial complex — a connect-the-dots skeleton of the cloud.
Security analogy: a graph where you draw an edge between two flows that are "similar enough."
Vietoris–Rips filtration
The key trick. Start with a distance threshold ε = 0 (no dots connected). Slowly grow ε. Any two dots closer than ε get linked. As ε grows, clumps merge and loops form and then fill in. A filtration is this whole growing movie of the shape, from fine detail to one big blob. "Vietoris–Rips" is just the rule for when to connect dots (by distance).
Security analogy: like tuning a similarity threshold from strict to loose and watching how the connection graph forms.
Homology dimensions: H0 and H1
Homology is the bookkeeping of shape. H0 = connected components (how many separate clumps). H1 = loops (how many 1-dimensional holes / rings). Barcode tracks both. Higher dimensions (H2 = voids) exist but aren't used here.
Security analogy: H0 ≈ "how many distinct behavior clusters"; H1 ≈ "are there cyclic/repeating interaction patterns."
Birth, death & persistence
As ε grows, a loop might appear ("born") at one threshold and disappear ("die") at a larger one. Persistence = death − birth: how long the feature survives. Long-lived features are real structure; short-lived ones are noise.
Security analogy: persistence is your signal-to-noise filter — keep the structure that survives, ignore the flicker.
Persistence diagram
The final summary: a scatter plot where each dot is one feature, plotted as (birth, death). Dots far from the diagonal = strong, long-lived structure. This single picture is the topological signature of a traffic window. (In the code, each diagram row is (birth, death, dimension).)
Security analogy: the "shape fingerprint" of a 50-flow window. Compare two windows by comparing their diagrams.
Vectorizing a diagram: persistence image, Betti curve, landscape
A diagram is a set of dots, which is awkward for standard ML. These three methods turn it into a fixed list of numbers: a persistence image (blur the dots into a heatmap), a Betti curve (count features vs. threshold), or a persistence landscape (stack the features into a profile). All three say the same thing in a model-friendly format.
Security analogy: turning a rich structural fingerprint into a feature vector your existing tooling can ingest.
Wasserstein distance (optimal transport)
How Barcode measures "how different are two shapes." Imagine each diagram as piles of sand; the Wasserstein distance is the minimum work to reshape one pile into the other (that's "optimal transport"). The order (here 2) just sets how heavily big moves are penalized. Crucially it is stable: a small change in traffic = a small change in distance, so it doesn't overreact to noise.
Security analogy: a principled "distance from normal." Bigger distance = more anomalous window = higher alert score.
Reference diagram
Barcode averages the diagrams of many BENIGN windows into one "what normal looks like" template. Every new window is scored by its Wasserstein distance (on H1 loops) to this reference.
Security analogy: your behavioral baseline. Everything is judged as deviation from it.
Threshold (95th percentile)
How far is "too far"? Barcode measures the distances of a held-out set of known-benign windows and takes the 95th percentile as the alarm line. Anything above it is flagged. By construction, that's about a 5% false-alarm rate on clean traffic — a knob you can tune.
Security analogy: exactly like setting a detection threshold to hit an acceptable false-positive budget.
Stability / robustness to noise
A formal property of persistent homology: small perturbations to the input move the diagram only a little. This is the mathematical reason the method tolerates the natural jitter in real traffic.
Security analogy: fewer false positives from benign variability; the detector reacts to structure, not jitter.

3 The four detectors

Barcode doesn't bet on one model. It runs several and compares them — and fuses the two most complementary ones.

3.1 TopoDetector (the star of the show)

The topology pipeline from Section 2, packaged as a detector: build the window's persistence diagram, measure its Wasserstein distance to the BENIGN reference, alert if above threshold. Fully unsupervised — it never sees attack labels during fitting, only normal traffic.

3.2 Isolation Forest (classic baseline)

An ensemble of 200 random trees that isolate points by random splits. Anomalies get isolated in few splits because they sit in sparse regions. contamination=0.05 tells it to expect ~5% outliers. It's a strong, fast, per-window baseline that ignores topology entirely — included to prove the topological method earns its keep.

3.3 Autoencoder (classic baseline)

A small neural network (64→32→16 then back out) trained only on benign traffic to compress and reconstruct it. Benign input reconstructs well (low error); anomalous input reconstructs badly (high error), and that reconstruction error is the anomaly score. Trained with MSE loss via Adam for 50 epochs.
Security analogy: "it learned normal so well that anything it can't redraw is suspicious."

3.4 GNN — GraphSAGE (the smart helper)

Some attacks barely change the topology but live in the relationships between flows. So Barcode also builds a graph from each window and uses a graph neural network.

  • Graph / node / edge: features are quantized into pseudo-endpoints (nodes); each flow becomes an edge connecting two endpoints. The window becomes a small network-of-flows.
  • GraphSAGE: a GNN that learns by message passing — each node updates itself by aggregating ("sampling and aggregating") information from its neighbors. Architecture: SAGEConv 64→32 + a small classifier head, trained as a supervised attack/benign classifier (BCE loss, 20 epochs).
Why two methods?
TDA is unsupervised and great at structural attacks; the GNN is supervised and great at relational/edge-level attacks (the quiet ones). They cover each other's blind spots.

3.5 Late fusion (combining the scores)

The Fused detector takes the topo score and the GNN score, puts both on a 0–1 scale (min-max normalization), and blends them with a weighted average (convex combination, weight α = 0.5 = equal trust). "Late" fusion = combine the two final scores rather than merging the models internally.
Security analogy: two analysts vote; you average their confidence into one alert score.

4 The anomalies: each attack and how it bends the shape

This is the part where the math pays off. Barcode is tested on BENIGN traffic plus these attack families from CICIDS2017. For each, here is the behavior and what it does to the topological shape.

AttackWhat it does on the wireHow it perturbs the shape
BENIGN
the baseline		 Normal mix of sessions: varied durations, balanced forward/backward packets, moderate rates. A broad, structured cloud with characteristic clumps (H0) and some loops (H1). This is the reference everything is compared to.
DDoS Floods: very high packet rate, tiny packets, extremely one-sided (many forward, almost no backward), tight timing. Collapses connected components. Thousands of near-identical flows pile onto the same spot — the cloud crushes into one dense blob, so H0 drops sharply. Big Wasserstein jump.
PortScan Probing many ports: ultra-short flows, tiny SYN-like packets, very few packets each. Breaks the loop structure. Normal request/response sessions form cyclic patterns (H1 loops); scans are one-shot, so those loops vanish and the cloud becomes spiky/scattered.
FTP-Patator
SSH-Patator
brute force		 Automated login guessing: many near-identical attempts with very regular, periodic inter-arrival times and consistent packet sizes. Over-regular, repetitive structure. The cloud becomes unnaturally tight and periodic compared to the natural spread of human sessions — a distinct, low-variance signature.
Web Attack —
Brute Force		 Repeated credential attempts over HTTP: longer flows, larger packets than benign web traffic. Shifts the cloud toward longer-duration, larger-packet regions and tightens it — a detectable drift plus reduced spread.
Infiltration
the hard case		 Stealthy, low-volume intrusion engineered to look almost exactly like normal traffic, flow by flow. Barely moves the shape. This is where pure topology struggles — the diagram is close to benign. The GNN rescues it using edge-level relationships the shape can't see. The whole reason fusion exists.
Bot Command-and-control beaconing: long-lived connections that "phone home" on a very regular schedule (highly periodic timing). Tight periodic clusters. The metronome-like timing produces a small, dense, repeating structure unlike the irregular spread of benign sessions.
The one mental model to keep
"Perturbation" here just means: the attack pushes the cloud away from its normal shape. Floods crush it (fewer components), scans de-loop it (fewer rings), brute-force/bots over-regularize it (too tight/periodic). Barcode scores exactly that push with the Wasserstein distance — and for the pushes too subtle to see, the GNN listens to the relationships instead.

5 The scoreboard: how "good" is measured

Every detector is graded on the test split, overall and per attack type. These are standard detection metrics — likely familiar, included for completeness.

MetricPlain meaning
ROC-AUCAcross all thresholds, the chance the detector ranks a real attack above a benign window. 1.0 = perfect, 0.5 = coin flip.
PrecisionOf everything it flagged, what fraction were truly attacks. High precision = few false alarms.
RecallOf all real attacks, what fraction it caught. High recall = few misses.
F1The balance (harmonic mean) of precision and recall in one number.
Average PrecisionPrecision–recall summarized across thresholds; especially honest when attacks are rare (imbalanced data).

6 The dataset & the stack

CICIDS2017

A widely used intrusion-detection benchmark from the Canadian Institute for Cybersecurity: realistic labeled network flows, BENIGN plus 9 attack categories. Barcode trains its baseline on 70% of benign flows, calibrates the threshold on 10%, and tests on the remaining 20% benign + all attacks. If the CSVs aren't present, the dashboard generates realistic synthetic traffic so it's always demoable.

Tools used

  • Python 3.11 — the language.

  • Streamlit + Plotly — the interactive dashboard.
TL;DR for your toolkit
Barcode is a behavior-based, mostly-unsupervised aggregate-traffic detector. Its novelty is scoring the geometric shape of traffic windows with persistent homology and an optimal-transport distance, backed by a graph neural network for the stealthy cases, and benchmarked honestly against classic anomaly detectors. Think of it as a complementary detection layer that triggers on structural anomalies signatures and per-flow models can miss.
